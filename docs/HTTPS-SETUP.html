HTTPS SETUP FOR BITCOINTORRENT

Setting up SSL on stream.bitcointorrent.online so the Vercel website can connect to the Hetzner streaming server without mixed content errors.

WHAT YOU NEED TO DO

Step 1: Add a DNS A record at your domain registrar (livedns.co.uk):

Type: A
Name: stream
Value: 135.181.103.181
TTL: 300

This points stream.bitcointorrent.online to your Hetzner server.

Step 2: SSH into Hetzner:

ssh hetzner

Step 3: Create the nginx config (needs sudo password):

sudo bash -c 'cat > /etc/nginx/sites-enabled/stream.bitcointorrent.online << "EOF"
server {
    listen 80;
    server_name stream.bitcointorrent.online;
    location / {
        proxy_pass http://127.0.0.1:8404;
        client_max_body_size 500M;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection upgrade;
        proxy_read_timeout 86400;
    }
}
EOF'

Step 4: Test and reload nginx:

sudo nginx -t && sudo nginx -s reload

Step 5: Get the SSL certificate (Let's Encrypt via Certbot):

sudo certbot --nginx -d stream.bitcointorrent.online --non-interactive --agree-tos -m hello@b0ase.com

Step 6: Test it works:

curl -s https://stream.bitcointorrent.online/api/catalog

You should see the JSON catalog with the 3 videos.

AFTER HTTPS IS WORKING

Update the Vercel web app (app.html) to use https://stream.bitcointorrent.online instead of http://135.181.103.181:8404. This fixes the mixed content error.

WHAT THIS DOES

Browser (HTTPS) → stream.bitcointorrent.online (HTTPS/443)
    → nginx reverse proxy
    → localhost:8404 (BitCoinTorrent Docker container)

Nginx handles SSL termination. The Docker container stays on HTTP internally. Certbot auto-renews the Let's Encrypt certificate.
